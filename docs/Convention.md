# Git Commit Convention

1. 기능을 추가한 경우
  ```bash
  git commit -m
  "feat: add project detail"
  ```

2. 버그를 수정한 경우
  ```bash
  git commit -m
  "fix: fix code version issue"
  ```

3. 코드를 리팩토링한 경우
  ```bash
  git commit -m
  "refactor: main routing api"
  ```

4. 코드 스타일을 변경한 경우
  ```bash
  git commit -m
  "style: change login api flow"
  ```

5. 문서 작업
  ```bash
  git commit -m
  "docs: update README.md"
  ```

6. 파일 혹은 폴더명을 수정하거나 옮긴 경우
  ```bash
  git commit -m
  "rename: rename files"
  ```
